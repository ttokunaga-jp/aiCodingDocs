# サービス憲章（Service Charter / Mission Doc）

目的: 「このマイクロサービスは何をし、何をしないか」を一枚で定義し、責務逸脱を防ぐ。

```markdown
サービス名: Order History Service
ビジネス目的: ユーザーの注文履歴を高速に提供し、CS対応とレコメンドの精度を向上させる
対象ユーザー: Webフロントエンド、CSツール (内製)、レコメンドバッチ
責務 (Must):
  - ユーザーIDから注文履歴を取得し返却する (過去2年分)
  - 期間フィルタとステータスフィルタで検索できる
  - レポーティング用に集計 API を提供する
非責務 (Must Not):
  - 決済状態の更新・キャンセル受付
  - 推論やレコメンド生成
成功条件 (SLO/SLI 概要):
  - 可用性: 99.9%
  - レイテンシ: p95 < 200ms @ 1k RPS
  - 正確性: 2年以内の履歴欠損 0%
```
